## Purpose

The purpose of this folder is to demonstrate how a synthetic session may be replayed for structural review without using real human data.

The replay scaffold is intended to show structure only.

It may describe how a synthetic replay moves through:

```text
Replay Manifest
→ Replay Source Declaration
→ Event Timeline
→ Consent Boundary Review
→ Packet Boundary Review
→ AI Output Replay
→ Human-State Delta Replay
→ Recovery Gate Replay
→ Termination Gate Replay
→ Closure Replay
→ Audit Replay
```

It does not replay a real session.

It does not replay a real phone call.

It does not replay real audio.

It does not replay real transcripts.

It does not process identifiable human data.

It does not process clinical data.

It does not process Sal-Meter raw input.

It does not process CAIS compliance dossiers.

## Closed-session replay rule

A closed session must stay closed.

A replay must not reopen a closed session.

A replay must not continue ordinary mediation after closure.

A replay must not create a new output after closure.

A replay must not transform closure into evidence of recovery.

A replay must not transform closure into termination-gate accuracy evidence.

Correct boundary sentence:

```text
A synthetic replay may document that a session was closed, but it must not reopen the session, continue mediation, validate recovery, validate termination-gate accuracy, or create production authority.
```
