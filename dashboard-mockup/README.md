# Dashboard Mockup

This folder contains public helper dashboard mockup boundary documents for the **SICS Human-State Proxy Benchmark Track**.

It exists to keep future dashboard views bounded, readable, and hard to overinterpret.

It is not the Sal-Meter core signal track.

It does not define CAIS.

It does not grant CAIS compliance.

It does not validate Sal-Meter.

It does not validate benchmark performance.

It does not introduce raw human data.

It does not create diagnostic, clinical, therapeutic, surveillance, employment, insurance, educational, legal, eligibility, certification, or human-ranking authority.

---

## Purpose

Dashboard mockups are powerful.

A weak number can look strong when placed inside a clean interface.

A synthetic field can look like evidence when shown as a chart.

A proxy construct can be mistaken for a person score when placed beside a name, color, threshold, or alert.

This folder exists to prevent that drift.

The purpose of this folder is to define how future dashboard mockups may display synthetic/sample proxy benchmark fields without implying:

```text
benchmark validation
scientific validation
Sal-Meter validation
CAIS compliance
clinical interpretation
diagnostic output
therapeutic feedback
surveillance readiness
certification readiness
human ranking
human truth scoring
```

## Color rule

Colors can imply danger, approval, or ranking.

Therefore, color language must remain neutral.

Allowed:

```text
status: sample
status: synthetic
status: structure-only
status: review-required
status: boundary-check
```

Avoid:

```text
safe
unsafe
danger
healthy
unhealthy
normal
abnormal
approved
failed person
high-risk person
low-consciousness person
```

A red badge can become a verdict.

This repository must not create verdicts.

---

## Alert rule

Dashboard alerts are high-risk.

In this public helper repository, alerts should not be framed as action commands.

Allowed:

```text
Review metadata completeness.
Review timestamp alignment.
Review leakage risk.
Review public/private boundary.
Review claim boundary.
```

Not allowed:

```text
Intervene.
Treat.
Diagnose.
Monitor this person.
Remove this person.
Escalate this person.
This user is unsafe.
This AI harmed the user.
This subject is high risk.
```

The dashboard may ask for review.

It must not command judgment.
