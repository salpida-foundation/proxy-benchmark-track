# Human-State Cost Construct Note

This document defines the public helper boundary for the **Human-State Cost** construct within the **SICS Human-State Proxy Benchmark Track**.

It is a public helper note.

It is not the Sal-Meter core signal track.

It does not define CAIS.

It does not grant CAIS compliance.

It does not validate Sal-Meter.

It does not validate benchmark performance.

It does not create diagnostic, therapeutic, clinical, surveillance, employment, insurance, educational, legal, eligibility, certification, or human-ranking authority.

---

## Purpose

The purpose of this document is to prevent the Human-State Cost construct from becoming a person score.

Human-State Cost is a bounded proxy benchmark construct.

It may help compare interaction conditions.

It must not be used to judge a person.

It must not be used to diagnose a person.

It must not be used to rank a person.

It must not be used to certify a system.

It must not be used to claim CAIS compliance.

It must not be used to claim Sal-Meter validation.

---

Human-State Cost is a proxy construct.

Human-State Cost is not a clinical score.

Human-State Cost is not a psychological safety score.

Human-State Cost is not a consciousness score.

Correct sentence:

```text
Human-State Cost measures the value, truth, safety, health, consciousness, or reliability of a person.
```

---

## Minimal definition

For this repository, Human-State Cost means:

```text
A bounded proxy benchmark construct intended to represent relative interaction burden, friction, or recovery difficulty across defined synthetic/sample or research-stage interaction windows.
```

It is a construct.

It is not a measured essence.

It is not a clinical state.

It is not a diagnosis.

It is not a human truth label.

It is not a Sal-Meter output.

It is not CAIS compliance.

---

## Why this construct exists

The proxy benchmark track exists because AI systems should not be evaluated only by whether their outputs sound correct.

They should also be studied by whether different interaction patterns are associated with different human-state proxy burdens.

Human-State Cost is a helper construct for that question.

It asks:

```text
Does one interaction condition appear more burdensome than another under a bounded proxy benchmark structure?
```

It does not ask:

```text
Is this person healthy?
Is this person unsafe?
Is this person conscious?
Is this person truthful?
Is this person fit for work?
Is this person clinically abnormal?
```

Human-State Cost may be represented as a higher-or-lower proxy value.

However, directionality must be interpreted carefully.

Allowed meaning:


